Parte 1 · El gremio

Datos, funciones y roster

Etapa 1 · El gremio nace Nivel 1

Todo empieza por los datos. El gremio es un OBJETO; su roster es un arreglo de héroes (también objetos).

objetoarreglo de objetospunto / corchete
  1. Copia el objeto gremio al inicio de tu gremio.js. Tiene un nombre y un arreglo heroes, donde cada héroe es un objeto con nombre, clase, nivel, oro y vida.
  2. Muestra el nombre del gremio (notación punto) y el nombre del primer héroe (arreglo + propiedad).
¿Necesitas una pista?
  • A una propiedad de un objeto llegas con el punto.
  • El primer elemento de un arreglo está en la posición 0; desde ahí, accedes a su propiedad.
  • Para armar texto con datos adentro, usa backticks y ${...}.
gremio.js — punto de partida
const gremio = {
  nombre: "Los Valientes",
  heroes: [
    { nombre: "Aragorn", clase: "Guerrero", nivel: 5, oro: 120, vida: 100 },
    { nombre: "Gandalf", clase: "Mago",     nivel: 8, oro: 200, vida: 80 },
    { nombre: "Legolas", clase: "Arquero",  nivel: 3, oro: 90,  vida: 0 },
  ],

  // En las próximas etapas agregarás métodos AQUÍ dentro.
};

// Muestra el nombre del gremio y el del primer héroe

Gremio: Los Valientes
Primer héroe: Aragorn

Legolas tiene vida 0 a propósito (está caído): te servirá más adelante para filtrar.

Etapa 2 · Funciones arrow utilitarias Nivel 2

Antes de los métodos, tres herramientas arrow que el sistema reutilizará. Una de ellas con valor por defecto.

arrow functionsfindternariovalor por defectoMath
  1. buscarHeroe(nombre): devuelve el héroe del roster cuyo nombre coincida (sin importar mayúsculas). Usa find.
  2. estadoVida(vida): arrow con un ternario que devuelve "💀 caído" si la vida es 0, o el texto de la vida si está vivo.
  3. tirarDado(max): devuelve un número al azar entre 10 y max + 9. El max debe tener valor por defecto 50.
¿Necesitas una pista?
  • find recorre un arreglo y devuelve el PRIMER elemento que cumple una condición. Para comparar nombres sin importar mayúsculas, pasa ambos textos a minúsculas antes de comparar (método toLowerCase).
  • estadoVida es de una sola línea: ideal para un ternario (condición ? valorA : valorB).
  • Para un número al azar: Math.random() da entre 0 y 1, y Math.floor lo baja a entero. El valor por defecto se escribe en el propio parámetro.
gremio.js — agrega debajo del objeto
// Crea 3 funciones arrow (van FUERA del objeto):
//
//   buscarHeroe(nombre) -> el héroe cuyo nombre coincida (sin importar mayúsculas)
//   estadoVida(vida)    -> "💀 caído" si la vida es 0, o el texto de la vida si está vivo
//   tirarDado(max)      -> un número al azar entre 10 y max+9 (max por defecto: 50)
//
// Después pruébalas con console.log.

Encontrado: Gandalf
Estado con 0 vida: 💀 caído
Estado con 100 vida: 100 vida
Dado: (un número entre 10 y 59, varía)

Etapa 3 · Ver el roster (this + for + destructuring) Nivel 3

El primer método. Recorre los héroes con un for clásico y saca sus datos con destructuring.

métodothisfor clásicodestructuring
  1. Agrega el método verRoster() que recorra this.heroes con un ciclo for e imprima la ficha de cada héroe.
  2. Dentro del for, usa destructuring para sacar nombre, clase, nivel, oro, vida del héroe.
  3. Para el estado, reutiliza tu función estadoVida(vida) de la etapa 2.
¿Necesitas una pista?
  • Es un método: defínelo con function (no arrow), para que this sea el gremio.
  • El for clásico usa un índice i que va de 0 hasta el largo del arreglo (.length).
  • Destructuring: puedes sacar varias propiedades de un objeto a variables de un golpe, escribiendo sus nombres entre llaves.
  • Para el estado, reutiliza tu función estadoVida de la etapa 2.
gremio.js — agrega DENTRO del objeto
// Agrega el método verRoster(). Debe:
//   - imprimir un título con el nombre del gremio (this.nombre)
//   - recorrer this.heroes con un ciclo for
//   - por cada héroe, sacar sus datos con destructuring y mostrar su ficha
//     (usa tu función estadoVida para el estado)
//
// Luego, fuera del objeto: gremio.verRoster();

Etapa 4 · Reclutar (valor por defecto) Nivel 2

Un método para sumar aventureros. Si no se indica la clase, entra como "Aventurero".

métodopushvalor por defecto
  1. Agrega reclutar(nombre, clase) que cree un héroe nuevo (nivel 1, 0 oro, 100 vida) y lo agregue con push.
  2. El parámetro clase debe tener valor por defecto "Aventurero".
  3. Prueba reclutando a "Gimli" el "Guerrero", y a "Sam" sin pasar clase (debe quedar como Aventurero).
¿Necesitas una pista?
  • Para agregar un elemento al final de un arreglo, ¿qué método conoces? (empieza con p...)
  • El nuevo héroe es un objeto: define sus propiedades fijas (nivel 1, oro 0, vida 100).
  • El valor por defecto se escribe en el propio parámetro de la función.
gremio.js — agrega DENTRO del objeto
// Agrega el método reclutar(nombre, clase). Debe:

Misiones con suerte

Etapa 5 · Enviar a una misión (Math) Nivel 3

La etapa estrella: la SUERTE decide. Aquí brillan Math.random y Math.max, y reutilizas tus funciones de la etapa 2.

Math.randomMath.maxcondicionalesguard clausescambiar estado
  1. Agrega enviarMision(nombreHeroe): busca el héroe con tu buscarHeroe. Si no existe o está caído (vida 0), avisa y termina (guard clauses).
  2. Tira la suerte con Math.random() < 0.6 (60% de éxito). Si gana: suma oro con tirarDado(50) y sube un nivel. Si pierde: réstale vida con tirarDado(30).
  3. Al restar vida, usa Math.max(0, ...) para que NUNCA quede negativa.
  4. Envía a alguien a la misión un par de veces y observa los cambios.
¿Necesitas una pista?
  • Para "salir temprano" cuando algo no es válido (no existe, está caído), usa un return dentro de un if (guard clause).
  • Math.random() da un número entre 0 y 1. ¿Cómo lo comparas para que el éxito ocurra el 60% de las veces?
  • Para sumar/restar a una propiedad: heroe.oro += ... o heroe.vida -= ....
  • Para que un valor no baje de un piso (0), existe Math.max.
gremio.js — agrega DENTRO del objeto
// Agrega el método enviarMision(nombreHeroe). Debe:
//   1) buscar el héroe (reutiliza buscarHeroe). Si no existe o está caído, avisar y terminar
//   2) decidir con la suerte si hay éxito (60% de probabilidad)
//   3) si gana: sumar oro (tirarDado) y subir un nivel
//      si pierde: restarle vida (tirarDado), evitando que quede negativa
//
// Prueba: gremio.enviarMision("Aragorn");  (varias veces, el azar cambia)

⚔️ Aragorn triunfó! +42 oro, ahora nivel 6
(o, con mala suerte:)
🩸 Aragorn fue herido (queda 82 vida)

Etapa 6 · El ranking (sort) Nivel 3

¿Quién es el más poderoso? Ordena el roster por nivel, de mayor a menor.

sortspread (copia)
  1. Agrega ranking() que muestre los héroes ordenados por nivel, de mayor a menor.
  2. Ordena con sort sobre una copia del arreglo (con spread), para no desordenar el roster original.
¿Necesitas una pista?
  • El método sort ordena un arreglo. Recibe una función que compara dos elementos (a, b).
  • Para copiar un arreglo ANTES de ordenarlo (y no desordenar el original), recuerda el operador spread [...].
  • ¿a.nivel - b.nivel o b.nivel - a.nivel para mayor a menor? Pruébalo y observa.
gremio.js — agrega DENTRO del objeto
// Agrega el método ranking(). Debe:
//   - mostrar los héroes ordenados por nivel, de MAYOR a menor
//   - ordenar sobre una COPIA del arreglo (no sobre el roster real)
//
// Fuera: gremio.ranking();

Etapa 7 · Tesoro y bajas (reduce + filter) Nivel 3

Las cuentas del gremio: cuánto oro juntaron entre todos, y quiénes siguen en pie.

reducefiltermap
  1. Agrega tesoro() que devuelva la suma del oro de todos los héroes (con reduce).
  2. Agrega vivos() que devuelva los héroes con vida mayor a 0 (con filter).
  3. Muestra el tesoro total y los nombres de los héroes vivos (con map).
¿Necesitas una pista?
  • Para convertir un arreglo en UN solo valor (la suma del oro), está reduce (arrastra un acumulador).
  • Para quedarte solo con los que cumplen una condición (vida > 0), está filter.
  • Para transformar la lista de vivos en solo sus nombres, está map.
gremio.js — agrega DENTRO del objeto
// Agrega dos métodos:
//   tesoro() -> devuelve la suma del oro de TODOS los héroes
//   vivos()  -> devuelve los héroes con vida mayor a 0
//
// Luego muestra el tesoro y los NOMBRES de los héroes vivos.

La sala del gremio (menú)

🏆 Jefe Final · El menú do/while Integrador

Conecta todo: un menú interactivo para administrar el gremio llamando a tus métodos, hasta cerrar las puertas.

do/whilepromptif / else if

Arma un menú con do/while que, según la opción del usuario, llame a un método del gremio. Se repite hasta Salir. Opciones:

  1. Ver roster · Reclutar (pide nombre y clase) · Enviar a misión (pide nombre)
  2. Ver ranking · Ver tesoro · Ficha de un héroe (pide nombre) · Salir

Requisito: el menú NO repite lógica; solo invoca los métodos que ya creaste. Para leer la opción usa prompt() (o la función pedir() simulada de clase para probar sin ventanitas).

¿Necesitas una pista?
  • do/while es ideal para menús: ejecuta el cuerpo y RECIÉN al final revisa si hay que repetir. Así el menú se ve al menos una vez.
  • Cada opción es una rama del if / else if (o usa switch) que llama a UN método tuyo.
  • Para reclutar, misión o ficha, primero pide los datos con prompt y pásalos al método.
  • prompt siempre devuelve texto, así que compara la opción como texto ("1", "2"...).
gremio.js — cierre del archivo
// Construye al final del archivo un menú con do/while. Debe:
//   - mostrar las opciones (Roster, Reclutar, Misión, Ranking, Tesoro, Ficha, Salir)

//   - según la opción, llamar al método correspondiente del gremio
//   - repetirse hasta que elijan Salir
//
// Importante: el menú solo INVOCA tus métodos, no repite su lógica.
