🐾 Mi Mascota Virtual

Construye tu propio Tamagotchi: una mascota que es un objeto con propiedades (hambre, felicidad, energía) y métodos que cambian su estado usando this. La armas por etapas en mascota.js, aplicando TODO lo de objetos: notación punto/corchete, métodos, Object.entries, spread y destructuring. Versionas con Git (un commit por etapa), subes a GitHub y publicas en GitHub Pages. NO es el sistema que hicimos en clase: te toca aplicar las herramientas a algo nuevo.

📜 Cómo se juega

  1. Primero arma tu proyecto (sección de abajo): carpeta, mascota.js, index.html y repositorio Git.
  2. Resuelve las etapas en orden: cada una agrega una propiedad o método a tu objeto mascota.
  3. La misión dice QUÉ lograr, no CÓMO. Si te atascas, abre la 💡 pista.
  4. Prueba en tu computador (Node o Live Server + consola F12) y compara con la salida esperada.
  5. Cuando una etapa funcione: haz un commit. Una etapa = un commit.
  6. Marca "Etapa cumplida" y mira subir tu progreso.

Crea tu proyecto, versiónalo con Git, súbelo a GitHub y publícalo en GitHub Pages.

1 Crea tu carpeta

mi-mascota/

├── README.md      ← describe tu proyecto

<!DOCTYPE html>
<html lang="es-CL">

Para probar: abre index.html con Live Server y mira la consola (F12). Si tienes Node, corre node mascota.js en la terminal.

3 Inicializa Git y commitea por etapa

git init
git add .
git commit -m "chore: estructura inicial del proyecto"
cada etapa que funcione = un commit con mensaje claro (feat: ...). NO dejes todo para un commit gigante al final. Revisa con git log --oneline.

4 Sube a GitHub y publica en Pages

Subir (terminal, recomendado)

git branch -M main
git remote add origin https://github.com/TU-USUARIO/mi-mascota.git
git push -u origin main

Publicar

En tu repo: Settings → Pages, elige rama main y carpeta / (root). Abre el enlace y revisa la consola (F12).

Parte 1 · La mascota

Crea el objeto y su primer método

Etapa 1 · Nace tu mascota Nivel 1

Todo parte por el objeto. Copia la mascota base y aprende a leer sus propiedades de las dos formas.

  1. Copia el objeto mascota al inicio de tu mascota.js. Tiene nombre, tipo y tres stats (hambre, felicidad, energía) del 0 al 100.
  2. Muestra el nombre usando notación punto y el tipo usando notación corchete.
  3. (Puedes cambiarle el nombre y el tipo a tu gusto.)
¿Necesitas una pista?
  • Punto: mascota.nombre. Corchete: mascota["tipo"].
  • Ambas dan lo mismo; el corchete sirve cuando la clave es texto o variable.
mascota.js — punto de partida
const mascota = {
  nombre: "Pelusa",
  tipo: "gato",
  hambre: 50,      // 0 = lleno, 100 = muerto de hambre
  felicidad: 50,   // 0 = triste, 100 = feliz
  energia: 50,     // 0 = agotado, 100 = lleno de energía

  // En las próximas etapas agregarás métodos AQUÍ dentro.
};

// Muestra el nombre (punto) y el tipo (corchete)

Nombre: Pelusa
Tipo: gato

El nombre y tipo los eliges tú; lo importante es usar punto para uno y corchete para el otro.

Al terminar: git add mascota.js → git commit -m "feat: etapa 1 - objeto mascota"

Etapa 2 · El método describir() y this Nivel 2

Tu primer método: una función DENTRO del objeto que lee sus propias propiedades con this.

  1. Agrega un método describir() dentro del objeto que devuelva un texto como: "Pelusa es un gato."
  2. Dentro del método, usa this.nombre y this.tipo (NO escribas los valores a mano).
  3. Invócalo y muestra el resultado.
¿Necesitas una pista?
  • Sintaxis de método: describir() { return ... } dentro de las llaves del objeto.
  • this es la propia mascota: this.nombre lee su nombre.
  • Para invocarlo: mascota.describir(). ¡NO uses arrow function para el método! (con arrow, this no sería la mascota).
mascota.js — agrega DENTRO del objeto
// Dentro de las llaves del objeto, después de los stats:
//   describir() {
//     return `${this.nombre} ...`;
//   },


// Fuera del objeto: invoca y muestra mascota.describir()

Pelusa es un gato.

Parte 2 · Cuídala

Métodos que cambian el estado

Etapa 3 · alimentar() (cambiar el estado) Nivel 2

Lo nuevo y potente: un método que MODIFICA una propiedad del propio objeto con this.

  1. Agrega alimentar(): debe bajar el hambre en 20 (comer quita hambre).
  2. El hambre nunca debe bajar de 0: usa Math.max para ponerle un piso.
  3. Que imprima cómo quedó. Prueba alimentando dos veces (desde hambre 50).
¿Necesitas una pista?
  • Modificar el estado: this.hambre = this.hambre - 20.
  • Para que no baje de 0: this.hambre = Math.max(0, this.hambre - 20) (¿recuerdas Math?).
  • Dentro del método puedes hacer console.log mostrando this.hambre.
mascota.js — agrega DENTRO del objeto

//     this.hambre = Math.max(0, this.hambre - 20);

//   },

Etapa 4 · jugar() y dormir() Nivel 3

Dos métodos más, que cambian varias propiedades a la vez. Con tope arriba (no pasar de 100).

  1. jugar(): sube la felicidad en 20 (tope 100) y baja la energía en 15 (piso 0). Jugar cansa.
  2. dormir(): sube la energía en 30 (tope 100).
  3. Prueba: juega una vez y luego duerme una vez (desde 50/50).
¿Necesitas una pista?
  • Para no pasar de 100: Math.min(100, this.felicidad + 20).
  • Para no bajar de 0: Math.max(0, this.energia - 15).
  • Un método puede cambiar VARIAS propiedades: jugar() toca felicidad y energía.

Recorrer y copiar el objeto

Etapa 5 · Ver todos los stats (Object.entries) Nivel 3

Mostrar el estado completo recorriendo el objeto, pero solo los stats numéricos (sin los métodos).

  1. Agrega un método verEstado() que recorra la mascota e imprima solo las propiedades numéricas (hambre, felicidad, energía), no el nombre, el tipo ni los métodos.
  2. Usa Object.entries y filter para quedarte solo con los valores que son números.
  3. Pruébalo.
¿Necesitas una pista?
  • Object.entries(this) te da pares [clave, valor] de TODA la mascota (incluidos métodos).
  • Filtra por tipo: .filter(([clave, valor]) => typeof valor === "number").
  • Luego recorre con .forEach(([clave, valor]) => console.log(clave + ": " + valor)).
mascota.js — agrega DENTRO del objeto

//     Object.entries(this)

//   },

Etapa 6 · Clon VIP (spread) y destructuring Nivel 3

Crear una COPIA de la mascota con algo extra, sin tocar la original. Y sacar datos con destructuring.

  1. Crea mascotaVIP copiando tu mascota con spread { ...mascota }, pero agrégale un accesorio y ponle la felicidad en 100.
  2. Muestra la felicidad de la VIP y comprueba que la mascota original NO cambió.
  3. Con destructuring, saca nombre y tipo de la mascota a dos variables y muéstralas.
¿Necesitas una pista?
  • Spread con cambios: const mascotaVIP = { ...mascota, accesorio: "corona", felicidad: 100 }.
  • El spread crea un objeto NUEVO: cambiar la VIP no afecta a la original.
  • Destructuring: const { nombre, tipo } = mascota;.

// const mascotaVIP = { ...mascota, accesorio: ..., felicidad: 100 };

Jefe Final

Un día con tu mascota

🏆 Jefe Final · El menú do/while Integrador

Conecta todo: un menú interactivo que cuida a tu mascota llamando a sus métodos, hasta que decidas salir.

Arma un menú con do/while que, según la opción del usuario, llame a un método de tu mascota. Se repite hasta elegir Salir. Opciones:

  1. Alimentar (llama mascota.alimentar()).
  2. Jugar (mascota.jugar()).
  3. Dormir (mascota.dormir()).
  4. Ver estado (mascota.verEstado()).
  5. Salir.

Requisito: el menú NO repite lógica; solo invoca los métodos que ya creaste. Para leer la opción usa prompt() (o la función pedir() simulada que viste en clase, para probar sin ventanitas).

¿Necesitas una pista?
  • Estructura: let opcion; do { ...menú; opcion = prompt("Opción:"); if/else if... } while (opcion !== "5");.
  • Usa do/while para que el menú aparezca al menos una vez.
  • Cada opción del if/else if llama a UN método: if (opcion === "1") mascota.alimentar();.
  • Para probar sin ventanitas, copia el patrón pedir() con entradas simuladas de la clase y, en tu PC real, cámbialo por prompt.
